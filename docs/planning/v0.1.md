### ownership isolation: per-user scoping in protected procedures
labels: `architecture,backend,ready,priority: high`
```markdown
## Description
make every protected procedure resolve the current user and scope data access to them, so no user can read or write another user's records.

## Acceptance Criteria
- [ ] `protectedProcedure` rejects unauthenticated calls
- [ ] reusable ownership-scoped query helper
- [ ] tests proving cross-user access is denied

## Out of Scope
- résumé tables themselves (data model issue)
```

### design résumé data model + drizzle schema
labels: `architecture,ready,priority: high`
```markdown
## Description
model the résumé as structured section types (summary, experience, education, skills, projects, custom) with ordered items, order + visibility as data, and the fields the standards lack (company/context line, remote/availability, ongoing "present"). pick the persistence shape (serialized doc vs normalized) for clean versioning, and keep prose behind an indirection so i18n can be switched on later.

## Acceptance Criteria
- [ ] schema covers the section types + ordered items with order + visibility as data
- [ ] extra fields present: company/context, remote/availability, ongoing role
- [ ] persistence shape chosen and written down, decided with versioning in mind
- [ ] migration applies

## Out of Scope
- versioning tables (versioning model issue)
- editing ui + api (later)
- turning i18n on (seam only)
```
